## Set APP_URL environment variable

On the sandbox EC2 instance, add to the environment:

```
APP_URL=https://sandbox.getalloro.com
```

Then restart the Node process.

### Why

Share links generated by `/api/checkup/share` use `APP_URL` to build URLs. Without it, the fallback is `https://app.getalloro.com` (production). At AAE, every prospect who shares their checkup results would send friends to a domain that may not serve the app.

### Verification

After setting:
```bash
curl -s https://sandbox.getalloro.com/api/checkup/geo
```
Should return JSON with `lat` and `lng`.

Then from the app: run a checkup, create an account, click share. The generated share URL should start with `https://sandbox.getalloro.com/checkup/shared/`.

## Verify n8n is running (PMS upload dependency)

The PMS upload pipeline sends data to `https://n8n.getalloro.com/webhook/parse-csv` for processing. If n8n is down, uploads silently fail -- the user sees "success" but referral intelligence never populates in the dashboard.

### Verification

```bash
curl -s -o /dev/null -w "%{http_code}" https://n8n.getalloro.com/webhook/parse-csv
```

Should return `200` or `405` (method not allowed for GET is fine -- means the endpoint exists).

If n8n is down or the webhook doesn't exist, PMS uploads will appear to work but produce no output. Prospects at AAE who upload their data would see nothing happen.

## Email sending for trial accounts (Decision needed)

The 7-day trial drip sequence (Day 1 welcome, Day 3 reply-trap, Day 5 referral ask, Day 6 payment prompt, Day 7 value summary) is blocked on sandbox because `ALLOW_EMAIL_SEND` is not set. This is intentional -- prevents sending to real people during dev.

But it means every AAE signup gets silence after account creation. No Day 1 email, no Monday brief, nothing. The trial experience that converts relies on those touchpoints.

### Options

A) **Enable on sandbox with safeguards.** Set `ALLOW_EMAIL_SEND=true` and add an email allowlist or date filter so only accounts created after April 14 receive emails. Risk: accidental sends to test accounts.

B) **Deploy to production before AAE.** Point the QR code at production. Email pipeline works there. Risk: production deployment pressure, but it's the real environment anyway.

C) **Accept no emails during AAE.** Rely on the checkup + dashboard experience alone. Prospects get the booth wow but the 7-day trial is dashboard-only. Risk: lower trial-to-paid conversion because there's no re-engagement.

### What I'd recommend

B is the real answer if the goal is a 7-day trial that converts AND inspires sharing. The email sequence is half the trial experience. Without it, we're asking prospects to remember to open the dashboard on their own for 7 days.
