## Fixes in Priority Order

### 1. CRITICAL: Fabricated activity counts (GQ-02)
**File:** src/jobs/mondayEmail.ts, ~line 438
**Bug:** When real activity counts are zero, code fabricates: `competitorCount || (snapshot.competitor_name ? 5 : 3)`. Shows "scanned 5 competitors" when it scanned zero.
**Fix:** Remove fabricated fallback. If counts are all zero, send clean-week email instead. Never fabricate content.

### 2. CRITICAL: Test orgs receive emails (GQ-04)
**File:** src/jobs/mondayEmail.ts, sendAllMondayEmails function (~line 583)
**Bug:** Query includes all orgs matching subscription_status='active' OR checkup_score IS NOT NULL. No filter for test/demo orgs. 20+ test orgs from signup bug could get emails.
**Fix:** Add filter to exclude test orgs. Check if organizations table has an `is_demo` or `is_test` column. If not, filter by a known list or add the column.

### 3. HIGH: Em-dash stripping (GQ-15)
**Bug:** Three leak vectors where em-dashes reach client inbox:
1. intelligence.finding events from Claude (pulled by getMostShareableFinding)
2. weekly_ranking_snapshots.bullets (generated by rankings_intelligence)
3. Any dynamic content injected into templates
**Fix:** Create utility function and apply before template injection:
```typescript
function stripEmDashes(text: string): string {
  return text.replace(/\u2014/g, ', ').replace(/\u2013/g, '-');
}
```
Apply to: findingHeadline, findingBody, competitorNote, all bullets, rankingUpdate. Apply in both MondayBriefEmail.ts and CleanWeekEmail.ts templates.

Also fix the comment em-dash on line 2 of mondayEmail.ts: change the long dash to a comma.

### 6. MEDIUM: System Conductor runs after send, not before (GQ-20)
**Bug:** recordAgentAction (which routes through System Conductor) is called AFTER sendMondayBriefEmail. The 7 quality gates never see the content before the client does.
**Fix:** Move the Conductor check BEFORE the send. Build the email content, run it through conductorGate, and only call sendMondayBriefEmail if cleared. If held, log the reason and create fallback notification.

### 7. MEDIUM: Wrap recordEmailOutcome in try/catch (GQ-17)
**File:** src/jobs/mondayEmail.ts, ~lines 544 and 198
**Bug:** If feedback_outcomes table doesn't exist, recordEmailOutcome throws. This could cause the function to return false even though the email sent successfully.
**Fix:** Wrap both call sites in try/catch. Feedback loop failure should never affect email delivery status.

### 8. MEDIUM: Biological-economic lens in fallback bullets (GQ-13)

**Fix:** When building fallback bullets, append: "That gap represents approximately $X in annual revenue at risk" using avgCaseValue from vocabulary config. Name the human need: "safety: your team's livelihood depends on market visibility."

### 9. LOW: founderLine parameter dead code

**Fix:** Either wire founderLine into the template (replacing the hardcoded sign-off) or remove the parameter from the interface and all call sites.

### 10. LOW: Timezone awareness (GQ-18)
